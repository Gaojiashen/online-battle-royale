## NPC设计格式

每个NPC一个md文件，结构固定：基础信息表 → 背景叙事 → 表层身份 → 真实欲望（三个问题）→ 可提供的资源（表格）→ 性格与行为倾向（表格）→ 对应剧情线方向 → 与其他NPC的关联

## 设计理念

- 玩家与NPC交互为固定选项（非自由对话）
- 选择支结构：场景描述 → 2-4个选项 → 不同结果
- NPC可以死亡，世界自行运转
