# Project_Constitute
这是一款受《密教模拟器》启发的多人在线文字沙盒游戏。

玩家在同一个动态世界中探索秘密、参与势力斗争、获取超自然知识并推进自己的飞升计划，最终争夺飞升资格。

玩家的最终目标是：完成飞升
所有的设计都应该服务于该目标

玩家应当体验：
探索未知
发现秘密
获得机会
争夺机会
推进飞升
暴露自身
遭遇竞争
接触更深层秘密
重复

玩家追逐的是秘密与飞升，而不是战斗本身。

核心循环：
探索世界
获取情报与秘密
发现飞升机会
采取行动争夺机会
获得知识、资源或影响力
推进飞升进程
引发新的风险与竞争
接触更深层秘密
重复

## 设计原则
1. 飞升优先

任何系统设计都应优先强化飞升体验。
当战斗体验与飞升体验发生冲突时，优先保留飞升体验。

2. 冲突是结果

玩家不应该为了战斗而战斗。
理想状态下，玩家因为争夺：
秘密
资源
仪式材料
NPC
势力利益
而产生冲突。
战斗只是解决冲突的方式之一。

3. 信息即资源

情报与知识应当具有极高价值。
玩家可以通过：
传闻
调查
人脉
文件
秘传
改变游戏进程。

4. 身份是入口

身份不是职业。
身份决定：
初始地点
初始人脉
信息渠道
特殊行动
不决定最终飞升路线。
任何身份都应具备飞升可能性。

5. 世界先于任务

世界应当拥有自己的运行逻辑。

优先设计：

势力
NPC
地点
事件

而不是任务列表。

玩家通过探索发现机会，而不是领取任务推进游戏。

6. 系统先于内容

设计顺序：

核心体验
→ 核心循环
→ 行动框架
→ 身份系统
→ 世界结构
→ 势力系统
→ 飞升系统
→ 战斗系统
→ 数值平衡
→ 内容填充

禁止直接堆积设定与内容。

7. 法官友好

任何机制都必须考虑法官执行成本。

优先采用：

标准化
可记录
可结算

的设计。

避免大量依赖临场裁定。

8. 可程序化

默认未来可能实现：

网页化
APP化
AI法官

因此设计时应尽量结构化。

行动、状态、资源、关系和结果都应能够数据化表示。

当前定位

本项目不是传统大逃杀。

本项目也不是传统TRPG。

本项目的本质是：

多人共享世界中的飞升竞赛。

9.信息可信原则

在本项目中，信息本身是一种资源。

玩家会主动分析：

NPC对话
环境描述
地点介绍
报纸内容
事件记录
传闻与线索

并据此制定行动计划。

因此，所有向玩家呈现的信息都应具有真实性和意义。

信息会创造玩家预期

当文本提及：

人物
地点
组织
事件
物品

时，玩家会自然认为其具有后续价值。

因此，被明确提及的内容应至少满足以下条件之一：

可互动
可调查
可验证
对世界观构建具有明确作用
禁止虚假线索

禁止为了增加氛围感而创造没有任何后续内容的信息。

例如：

提及不存在的NPC
提及无法到达的地点
提及没有任何意义的事件
提及永远不会出现的组织
提及没有用途的特殊物品

如果某个信息会引导玩家投入行动进行探索，则该探索应当获得合理回报。

奖励好奇心

当玩家注意到某个细节并选择调查时，应当获得价值反馈。

反馈形式可以包括：

情报
资源
剧情
NPC
世界观知识
新的探索方向

玩家不应因为认真阅读和主动探索而发现世界内容为空。

核心要求

要么实装，要么不要提及。

任何被明确写入游戏文本的重要信息，都应当被视为世界内容的一部分，而非单纯的氛围描写。
