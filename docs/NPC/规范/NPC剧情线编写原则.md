# NPC 剧情线编写原则

> 基于伊索贝尔·格雷 & 马库斯·布莱克伍德两条剧情线迭代中沉淀的规则。
> 每条原则都附「为什么」和「反例 → 正例」。

---

## 一、核心约束（不可违反）

### 1. 玩家获得栏只放具体物品

**——** 情报通过选项结果对话自然传达，不单独列在"玩家获得"里。

❌ 反例：
```
| A | 问细节 | 他说了很多 | 情报：手记作者是洛克哈特 |
```

✅ 正例：
```
| A | 问细节 | 他沉默了一会儿：「那本手记的作者——署名是E·洛克哈特。」 | — |
```

**为什么**：列在"获得"栏里等于告诉法官"玩家记住了这个"——但玩家能不能记住、怎么用，应该由玩家自己决定。法官只需要把结果对话发出去。

### 2. 不设计暗中参数

—— 没有信任值、好感度、警惕值、危险等级。

**为什么**：暗参数法官看不见（在运营12个人时不可能手动算），玩家更看不见。保持规则透明：「选A→走到A方向，选B→走到B方向」。

❌ 「你引起了她的警觉，好感-1」
✅ 「她后退半步，语气变冷。她记住你的声音了——但在公共场合相遇，她会对你保持距离。」

区别：后者描述了**具体会发生什么行为**，不是在评分。

### 3. 选项结果必须有差异化

**为什么**：如果选A和选B看到的信息几乎一样，选项就没有意义。本游戏没有信任度/好感度等数值系统，纯对话风味的差异对玩家而言是虚假选择。

每条选项应该导向**不同的物质结果**：
- 获得不同的物品（羊皮纸 vs 蜡烛 vs 衣扣）
- 开启不同的后续路径
- 承担不同的风险（安全获取 vs 冒险获取更好物品）
- 消耗不同的资源（时间、金钱、体力）

❌ 选项A和B都能获得同一个情报，只是"氛围不同"
✅ 选项A→获得衣扣；选项B→获得地下草图；选项C→不搜索直接走

#### 3a. 仅传递信息的节点不设假选项

如果一个节点的目的只是向玩家传递剧情信息，且无论如何都会导向同一个出口——**不要强行拆分出多个选项来制造"有选择"的错觉**。

正确做法：
- 合并为**单一叙事事件**，标注时间消耗（如"花费0.5h"）
- 玩家投入时间 → 获得全部信息 → 自然推进到下一节点
- 只有当不同选项会导向**不同的物质结果**时，才设置选项

❌ 反例（来自马库斯原版节点1）：
```
选项A「听说这里教格斗」→ 打沙袋 → 出口："明天来办手续"
选项B「找马库斯」→ 简短对话 → 出口："打两组再走"
选项C「环顾四周不说话」→ 他继续打沙袋 → 出口："参观完了自己出去"
→ 三条路径同一个出口暗示，玩家没有获得任何不同的东西
```

✅ 正例（修改后）：
```
【单一事件】
场景：你推门进入黑铁俱乐部，马库斯在打沙袋。他看了你一眼。
事件：你花了约半小时打拳。他偶尔看你一眼，没多说话。
出口（离开时背后传来马库斯的声音）：
「左数第三个沙袋。俱乐部每天下午5:30开，晚上8:00打烊——打烊后是另一桌。」
→ 玩家花费0.5h，获得下一节点的入场条件，叙事自然推进
```

**判断标准**：问自己——"选A的玩家和选B的玩家，一小时后手里拿的东西、能去的地方、面临的风险，有任何不同吗？"如果没有，就不需要选项。

### 4. 出口暗示必须包含明确时间和地点

**为什么**：玩家需要知道"下一次在什么时候、什么地点"能找到NPC。模糊的"傍晚""过几天""下次"会导致玩家不知道如何推进。

❌
```
「修道院街的馅饼摊，傍晚的肉馅饼还不错。」
「西区有个旧仓库，去那里看看。」
```

✅
```
「修道院街的馅饼摊。下午5:00出摊，5:30就收摊了——排队的人很多，但味道确实值得等。」
「格拉斯市场西北角那条没人走的巷子——明天下午14:00，阳光刚好能照到活板门的位置。」
```

**时间表达用相对时间，不用周几。** 「明天下午和我一起来」而不是「每周二下午」。追踪周几对法官和玩家都是额外负担。

### 4a. 「搜索」是通用行动，不写进选项

**为什么**：搜索是玩家在每个地点都可以进行的通用行动。把「搜索地面层」写进选项等于暗示"你只能在这里搜索"——限制了玩家的行动空间。

在场景描述中可以暗示该区域有值得搜索的东西（"地面散落着木箱碎片和发黄的纸页"），但不把搜索本身作为选项。选项只涉及需要决策的分支行为。

❌ 选项：「A. 搜索地面层」「B. 直接下活板门」
✅ 场景中提到可被发现的元素，选项涉及的是"怎么做"而非"搜不搜"

## 二、结构规则

### 5. 链式推进，NPC只前不进

- 每个节点的出口对话中，NPC 暗示下一个互动地点/时间
- 节点推进后**不可重来**，不可回退到已完成节点
- 玩家A在节点④放弃 → 锁释放 → 玩家B从NPC**当前所在的节点**接手，不是从头开始

### 5a. 错过时间节点 = 自动放弃NPC剧情线

如果节点出口给出了明确的时间窗口（如"明天下午14:00"），而玩家未在约定时间出现：

- **该玩家自动放弃此NPC剧情线**，🔓锁释放
- **NPC停留在该节点位置**，以同样方式（同一时间、同一地点）等待下一名玩家
- NPC不会自己推进，不会因为"玩家没来"而改变剧情状态

❌ 玩家错过时间 → NPC自己推进了 → 但该玩家还能在别的时间找到他继续
✅ 玩家错过约定时间 → 🔓锁释放 → 马库斯每天14:00-15:00在殡仪馆附近等待 → 下一位玩家可从当前节点接手

**为什么**：① 给玩家明确的后果——"去不去"是一个有分量的决定；② 法官不需要追踪"谁在什么时间该出现在哪里"；③ NPC状态对全局透明。

### 6. 锁系统（全局共享）

每个有剧情线的NPC有四种锁状态：

| 状态 | 含义 |
|:----:|:------|
| 🔓 **无主** | 无玩家在推进，任意玩家首次交互即锁定 |
| 🔒 **锁定** | 某玩家在推进，只有该玩家可触发剧情节点 |
| ✅ **完结** | 剧情正常结束 |
| 💀 **终结** | NPC被击杀/绑架，所有玩家线关闭 |

**锁定状态下其他玩家的权限：**
- 推进剧情 → ❌（她拒绝："我在等一个人"）
- 观察位置/跟踪 → ✅ 可调查她的活动
- 攻击/绑架 → ✅ 成功则💀终结
- 拦截信息 → ✅ 可截获锁定玩家与NPC的通信

### 7. 节点数量指南

- 有剧情线的NPC：**5-7个节点**（含分歧点和终局）
- 无完整剧情线的NPC：**2-3个触发事件**（不设锁系统，不锁死）
- 关键分歧点建议放在第4-5个节点（玩家已有足够投入，放弃有成本）

### 8. 判定骰子使用

分歧节点可引入判定：

| 类型 | DC | 失败后果 |
|:-----|:---:|:---------|
| 协作行动（陪NPC） | 5-7 | NPC受伤/物品降级，但不死亡 |
| 远程协助（NPC独自行动） | 10-12 | NPC可能重伤/死亡 |

**原则**：判定失败≠剧情终结，除非玩家主动选择放弃。

## 三、叙事规则

### 9. 场景描述 = 可直接发给玩家的文本

场景描述框里的内容就是法官复制发送给玩家的。不要写法官速查摘要。

❌ 法官速查摘要：
```
入口节点。她在窗边读书，感知到你拿书的方向。
```

✅ 玩家阅读的场景：
```
图书馆靠窗的位置坐着一个少女。她面前摊着一本书，但没有在读——
她的手指安静地搁在书页上，像在感受纸面的温度。

她的眼睛覆着一层灰白色的翳膜。她微微侧了一下头，朝向你的方向。

「你拿的那本书——第三卷左边那本——是1542年版的《自然哲学纲要》。」
她轻声说，「印刷质量不太好，但第八章关于视觉的论述很有意思。」
```

### 10. 选项文本 = 玩家看到的选项

每个选项就是玩家可以选的文字，保持简洁，2-20字为宜。

❌ 太长："排到她身后，让她通过脚步声和气息感知到你的存在，然后等她买完后和她一起离开馅饼街"
✅ "排到她身后让她感知到你在"

### 11. 结果文本 = 可直接发给玩家的叙事

- 包含：发生了什么、玩家看到了什么/听到了什么、NPC的言语和反应
- **不要包含**：判定数值（DC6、骰子结果等）——这些只写在hint里给法官看

### 12. 放弃选项

关键分歧点和终局节点应提供"放弃/退出"选项，后果明确：
- 放弃 = NPC可能独自行动 → 可能受伤/死亡 → 锁释放
- 放弃后NPC不会有特殊反应（不恨你、不追杀你），玩家只是"没有再来的人"

## 四、其他设计原则（来自核心设计原则文档）

### 13. 不依赖日历
节点由玩家行动触发，不是固定在第X天发生

### 14. NPC可以死亡
不是所有NPC都要活到终局。世界自行运转。死亡是剧情的一部分

### 15. 预留NPC间交汇点

在设计剧情线时，留出一两个节点与其他NPC的关联：
```
节点⑤ → 蛾之教会同时在监视图书馆（与埃德蒙·里德线交汇）
```

这让不同剧情线的玩家可能在操作中产生交集。

## 五、速查清单（法官设计新剧情线时逐项核对）

- [ ] 每个节点的「玩家获得」栏只有具体物品（或无），没有情报类描述
- [ ] 没有信任值/好感度/警惕值等暗参数
- [ ] 同节点的不同选项导向**不同物质结果**（不同物品/路径/风险），无差异则合并为单一事件
- [ ] 「搜索」不作为选项，搜索是通用行动
- [ ] 每个出口有**明确时间+地点**，用相对时间不用周几
- [ ] 时间窗口节点标注了错过后果（放弃+npc留节点）
- [ ] 场景描述是完整的玩家可读文本（不只是给法官看的摘要）
- [ ] 结果文本是完整的玩家可读叙事（不只是给法官看的速查）
- [ ] 判定/数值信息只出现在hint或表格字段中，不出现在结果文本里
- [ ] NPC有锁状态转换路径（无主→锁定→完结/终结）
- [ ] 分歧节点提供了"放弃"选项及后果
- [ ] 与其他NPC至少有一个交汇点
