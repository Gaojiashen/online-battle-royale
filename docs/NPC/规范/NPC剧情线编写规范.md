# NPC 剧情线编写规范

## 概述

本文档定义 NPC 剧情线的数据结构和编写规则，确保所有 NPC 剧情线能正确导入管理工具并使用选项面板功能。

---

## 一、剧情线结构

一条完整的 NPC 剧情线由 **链式节点** 组成：

```
节点① → 节点② → 节点③ → ... → 终局
```

### 核心原则

- **链式推进**：每个节点的出口对话中，NPC 暗示下一个互动地点/时间
- **不依赖日历**：节点推进由玩家行动决定，不是现实日期
- **不追踪信任值**：节点走到哪就是哪，没有暗中参数
- **NPC 只前不进**：一旦推进到节点 N，不会回退到节点 N-1

---

## 二、节点数据结构

每个节点包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|:-----|:-----|:----:|:------|
| `node_number` | int | ✅ | 节点序号（从1开始） |
| `title` | str | ✅ | 节点名称（如"圣吉尔斯图书馆"） |
| `location` | str | ✅ | 具体位置（含地图区域编号，如"③圣吉尔斯图书馆 主厅"） |
| `description` | str | ✅ | **法官向玩家展示的场景描述**。这是玩家看到的内容，可直接复制发送 |
| `hint` | str | 可选 | 法官速查提示：下一节点入口线索（如"⤵ 修道院街馅饼摊，下午5:00"） |

每个节点包含 **2-4 个选项**：

| 字段 | 类型 | 必填 | 说明 |
|:-----|:-----|:----:|:------|
| `option_key` | str | ✅ | 选项编号（A/B/C/D） |
| `option_text` | str | ✅ | **玩家看到的选项文本**（如"在她对面坐下，翻开那本书"） |
| `result_text` | str | ✅ | **玩家选择后发生的结果叙事**，法官复制发送给玩家 |
| `player_gain` | str | 可选 | 玩家实际获得的东西（如"从今以后她凭脚步声认出你"）。**仅放具体可用的东西**，情报类由 result_text 体现 |

## 四、编写规则

### 4.1 时间表达

❌ **不要用**：傍晚、明天下午、过几天、不久后

✅ **要用**：下午5:00出摊、明天下午2:00、后天下午

### 4.2 出口暗示

每个节点的最后一句对话（或节点推进后的提示）应包含**明确的下一个地点+时间**，让玩家知道该去哪里、什么时候去。

```
节点① 出口：
「修道院街的馅饼摊。下午5:00出摊，5:30就收摊了——排队的人很多，但味道确实值得等。」
```

### 4.3 选项设计

- 每个节点 **至少 2 个选项，最多 4 个**
- 各选项应导向**差异化结果**——选 A 和选 B 应该得到不同的信息/物品/关系
- 不要有"暗中扣分"的选项
- 允许一个选项是"放弃/退出"（标记为 `option_key: "放弃"`）

### 4.4 结果文本

- 结果文本就是**法官可以直接复制发送给玩家的叙事**
- 包含：发生了什么、玩家看到了什么/听到了什么、NPC的反应
- 不要包含判定数值（如 DC 6）——这些写在法官速查的 `hint` 里

### 4.5 分歧与死亡

关键分歧节点（如伊索贝尔节点④）允许出现 **NPC 死亡**：

- **放弃/判定失败** → 在对应选项的 `result_text` 中写明死亡结局
- 死亡节点应标注在 `hint` 中（如 `"⤵ 陪她→解锁⑤；失败/放弃→💀"`）
- 死亡会触发 `lock_status = 终结`，所有玩家剧情线关闭

## 五、添加新 NPC 的步骤

### 5.1 在 database.py 中添加数据

1. 在 NPC 种子列表中添加 NPC 基本信息：
```python

```

2. 如果该 NPC 有剧情线，在 `NPC_NODES` 中添加节点数据：
```python

    ...
    "新NPC姓名": [
        {"num": 1, "title": "节点一", "loc": "区域", "desc": "描述", "hint": "提示"},
```

3. 在 `_seed_options()` 中添加选项数据：

```python
nodes_map = {}
```

## 六、常见设计模式

### 信息获取节点
```
选项 A → 套出关键情报（直接推进主线）
选项 B → 建立信任（后续获得额外线索）
选项 C → 保持距离（无收获但不损失）
```

### 风险决策节点
```
选项 A → 高风险高回报（需判定）
选项 B → 安全但收益较低
选项 C → 退出/放弃（可能触发NPC死亡）
```

### 资源交换节点
```
选项 A → 用情报换情报
选项 B → 用物品换服务
选项 C → 拒绝交易（下次来需要更高代价）
```
