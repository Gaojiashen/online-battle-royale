# 玩家战斗客户端设计文档

## 1. 架构概览

```
┌──────────────────────────────────────────┐
│              玩家浏览器                   │
│                                          │
│  ┌────────────────────────────────────┐  │
│  │        /player （HTML 单页面）       │  │
│  │                                    │  │
│  │  输入名称 → 看到战斗 → 选牌 → 出牌  │  │
│  └──────────────┬─────────────────────┘  │
│                 │                         │
│        ┌────────┴────────┐               │
│        │   FastAPI Server │               │
│        │                 │               │
│        │  GET /player    │ ← 页面HTML     │
│        │  GET /api/player/*│ ← 数据查询    │
│        │  POST /api/player/*│ ← 玩家操作   │
│        └───────┬─────────┘               │
│                │                          │
│        ┌───────┴─────────┐               │
│        │  飞书 Base       │               │
│        │                  │               │
│        │  对战管理         │ ← 战斗状态    │
│        │  玩家可用牌       │ ← 卡牌数据    │
│        │  回合提交         │ ← 出牌记录    │
│        │  对战记录         │ ← 结算日志    │
│        │  玩家战斗状态     │ ← HP/资源     │
│        └──────────────────┘               │
└──────────────────────────────────────────┘
```

> **核心原则：** 玩家页面调用 FastAPI → FastAPI 读写 Base。玩家不直接接触 Base token 和表结构。

## 2. 页面布局

### 2.1 未加入战斗状态（入口）

```
┌─────────────────────────────────────────┐
│  ⚔ 密教模拟器 S2 — 战斗仪表盘           │
│                                         │
│  ┌───────────────────────┐              │
│  │                       │              │
│  │   输入你的玩家名称      │              │
│  │   [_______________]    │              │
│  │                       │              │
│  │       [进入战斗]       │              │
│  │                       │              │
│  └───────────────────────┘              │
└─────────────────────────────────────────┘
```

- 玩家输入名称 → 调用 `/api/player/lookup?name=xxx`
- 后端查 Base「玩家」表 → 返回性相等级 + 当前对战ID
- 如果无对战 → 显示「暂无进行中的战斗」

- 点击「确认牌库」→ 调用 API → 等待对手确认

## 3. 玩家操作流程

### 3.1 完整时序

```
法官                        玩家A                    玩家B                FastAPI
 │                           │                        │                    │
 │  发起战斗                  │                        │                    │
 ├────────────────────────────────────────────────────────────────────────→│
 │  POST init-from-base      │                        │                    │
 │                           │                        │         写入Base    │
 │                           │                        │        玩家可用牌   │
 │                           │                        │                    │
 │                           │  打开 /player           │                    │
 │                           │  输入名称"高"          │                    │
 │                           ├───────────────────────→│                    │
 │                           │  GET lookup?name=高    │                    │
 │                           │  ← 性相+对战ID         │                    │
 │                           │                        │                    │
 │                           │  GET my-cards          │                    │
 │                           │  ← 可用卡牌列表        │                    │
 │                           │                        │                    │
 │                           │  勾选8张牌             │                    │
 │                           ├───────────────────────→│                    │
 │                           │  POST select-deck      │                    │
 │                           │  ← ok, waiting         │                    │
 │                           │                        │                    │
 │                           │                        │  同上流程            │
 │                           │                        │  选8张牌            │
 │                           │                        ├───────────────────→│
 │                           │                        │  POST select-deck  │
 │                           │                        │  ← 双方确认，对战开始│
 │                           │                        │                    │
 │                           │  ← 页面刷新，进入对战   │                    │
 │                           │                        │                    │
 │  ═══════════ 对战开始 ════════════════════════════                       │
 │                           │                        │                    │
 │                           │  Round N: 下拉选牌      │                    │
 │                           ├───────────────────────→│                    │
 │                           │  POST submit-card      │                    │
 │                           │  ← 提交成功，等待对手    │                    │
 │                           │                        │                    │
 │                           │                        │  同上提交           │
 │                           │                        ├───────────────────→│
 │                           │                        │                    │
 │                           │              ← 双方已提交 → 结算！           │
 │                           │                        │                    │
 │                           │  ← 页面刷新 → 看结果    │                    │
 │                           │                        │                    │
 │                           │  Round N+1...          │                    │
 │                           │                        │                    │
 │  ═══════════ 战斗结束 ════════════════════════════                       │
└───────────────────────────────────────────────────────────────────────────┘
```

### 3.2 状态机

```
                    ┌──────────┐
                    │ 无战斗    │
                    └────┬─────┘
                         │ 法官发起 → FastAPI写入可用牌
                    ┌────▼─────┐
                    │ 选牌中    │ ← 玩家看到可用卡牌
                    └────┬─────┘
                         │ 双方都确认8张 → FastAPI检验→锁定牌库
                    ┌────▼─────┐
                    │ 对战中    │ ← 每回合：选牌→提交→等对手→结算→下一回合
                    │  (循环)   │
                    └────┬─────┘
                         │ 一方HP≤0 或 双方同意结束
                    ┌────▼─────┐
                    │ 已结束    │ ← 显示胜者+全部日志
                    └──────────┘
```

## 4. API 设计

### 4.1 复用现有 API

| 端点 | 用途 | 调用方 |
|------|------|------|
| `GET /health` | 健康检查 | 页面启动时 ping |
| `POST /api/battle/init-from-base` | 法官发起战斗 | 法官面板 |
| `POST /api/battle/confirm-from-base` | 牌库确认（Base workflow 触发） | Workflow |
| `POST /api/battle/webhook` | 出牌提交（Base workflow 触发） | Workflow |
| `GET /api/battle/{battle_id}/status` | 战斗状态快照 | 可复用，但需扩展资源字段 |
| `GET /api/battle/{battle_id}/history` | 战斗日志 | 玩家页面直接调用 |

### 4.2 新增 API

#### `GET /api/player/lookup`

玩家输入名称后，查找自己在 Base 中的信息和当前对战。

```
请求：GET /api/player/lookup?name=高

响应：
{
  "ok": true,
  "player_name": "高",
  "aspects": {"灯": 4, "蛾": 6, "铸": 2, "冬": 2, "心": 3, "刃": 5},
  "game_hp": 82,
  "has_battle": true,
  "battle_id": "battle_abc123",
  "battle_state": "选牌中"
}
```

**后端逻辑：**
1. 查 Base「玩家」表 → 获取性相等级 + `当前对战ID`
2. 如果 `当前对战ID` 非空 → 查「对战管理」表 → 获取状态
3. 返回汇总结果

**Base 表改动：** 无。使用现有「玩家」表 + 「对战管理」表。

---

#### `GET /api/player/{player_name}/battle`

获取玩家当前战斗的完整信息（单人视角）。

```
请求：GET /api/player/高/battle

响应：
{
  "ok": true,
  "battle_id": "battle_abc123",
  "state": "对战中",
  "current_round": 3,
  "my_side": "A",
  "opponent_name": "橙子",
  "my_hp": 16,
  "opponent_hp": 12,
  "my_resources": {
    "edge": 2, "phantom": 0, "charge": 1,
    "chill": 0, "pulse": 3, "insight": 1
  },
  "my_deck": ["C01","B01","B03","W01","C03","B05","B06","C05"],
  "deck_confirmed": true,
  "my_submitted_this_round": false,
  "winner": null
}
```

**后端逻辑：**
1. 查「对战管理」→ 获取回合数、状态
2. 查「玩家战斗状态」→ 获取 HP + 六资源
3. 查「玩家」表 → 确定当前玩家是对战的 A 侧还是 B 侧
4. 查「回合提交」→ 检查本回合是否已提交
5. 返回汇总（**只返回当前玩家的信息，不暴露对手牌库**）

**Base 表改动：** 无。跨表查询聚合。

---

#### `GET /api/player/{player_name}/available-cards`

获取当前玩家的可用卡牌列表。

```
请求：GET /api/player/高/available-cards

响应：
{
  "ok": true,
  "battle_id": "battle_abc123",
  "cards": [
    {
      "card_id": "C01",
      "name": "挥击",
      "category": "进攻",
      "aspect": "通用",
      "level_requirement": 0,
      "effect_text": "造成2伤害",
      "selected": true
    },
    ...所有卡牌...
  ],
  "selected_count": 3,
  "deck_size": 8,
  "deck_locked": false
}
```

**后端逻辑：**
1. 查「玩家可用牌」→ 过滤 `对战ID` + `玩家侧`
2. 如果牌库已确认 → `deck_locked: true`，`selected` 反映已确认的8张

**Base 表改动：** 无。使用现有「玩家可用牌」表。

---

#### `POST /api/player/select-deck`

玩家确认 8 张牌的选择。

```
请求：POST /api/player/select-deck
{
  "player_name": "高",
  "battle_id": "battle_abc123",
  "card_ids": ["C01","B01","B03","W01","C03","B05","B06","C05"]
}

响应（等待对手）：
{
  "ok": true,
  "status": "waiting_for_opponent",
  "message": "已记录你的牌库，等待对手确认"
}

响应（双方确认）：
{
  "ok": true,
  "status": "battle_started",
  "current_round": 1,
  "message": "双方牌库已确认，对战开始！"
}
```

**后端逻辑：**
1. 验证 `card_ids` 长度 == 8
2. 验证全部卡牌都在该玩家的可用牌中
3. 写入「玩家战斗状态」的 `牌位1-8` 字段 + 标记 `牌库已确认 = true`
4. 检查对手是否也已确认 → 如果双方都确认，内部调用 `battle_manager.confirm_deck()`
5. 返回状态

**安全性：**
- 后端校验卡牌合法性（不可选自己不可用的牌）
- 不接受改牌库（`牌库已确认 = true` 后拒绝重复请求）

**Base 表改动：** 无。使用现有「玩家战斗状态」表的 `牌位1-8` + `牌库已确认` 字段。

---

#### `POST /api/player/submit-card`

玩家每回合提交一张出牌。

```
请求：POST /api/player/submit-card
{
  "player_name": "高",
  "battle_id": "battle_abc123",
  "round_number": 3,
  "card_id": "B03"
}

响应（等待对手）：
{
  "ok": true,
  "status": "waiting_for_opponent",
  "message": "已提交，等待对手出牌"
}

响应（立即结算）：
{
  "ok": true,
  "status": "resolved",
  "round": 3,
  "result": {
    "my_card": "连斩",
    "opponent_card": "佯攻",
    "rps_description": "连斩(进攻) vs 佯攻(佯攻) → 互伤",
    "damage_to_me": 2,
    "damage_to_opponent": 3,
    "my_hp_after": 14,
    "opponent_hp_after": 9,
    "special_events": ["锋芒+1", "脉动+1"]
  }
}
```

**后端逻辑：**
1. 验证卡牌在玩家本场牌库中
2. 在「回合提交」表中创建新记录
3. 检查对手是否也已提交本回合
4. 如果双方都已提交 → 调用 `battle_manager.submit_card()` → 触发 RPS 结算 → 写回 Base
5. 如果对手未提交 → 返回等待

**安全性：**
- 校验回合编号（不接受跳回合）
- 校验同一回合不可重复提交
- 校验卡牌合法性

**Base 表改动：** 无。使用现有「回合提交」表。

---

#### `GET /api/player/{player_name}/battle-logs`

获取战斗日志（只返回已结算的回合）。

```
请求：GET /api/player/高/battle-logs

响应：
{
  "ok": true,
  "battle_id": "battle_abc123",
  "logs": [
    {
      "round": 1,
      "my_card": "挥击",
      "opponent_card": "格挡",
      "rps_description": "挥击(进攻) vs 格挡(防御) → B看破+1",
      "damage_to_me": 0,
      "damage_to_opponent": 0,
      "my_hp_after": 20,
      "opponent_hp_after": 20
    },
    ...
  ]
}
```

**后端逻辑：** 查「对战记录」表 → 过滤 `对战ID` → 按回合号排序 → 返回。

> 也可以复用现有 `GET /api/battle/{battle_id}/history`，但因为返回的是双方全量信息，需要前端根据 `my_side` 做映射。新增此端点是为了**返回玩家视角的数据**（`my_card` / `opponent_card` 而非 `A使用卡牌` / `B使用卡牌`）。

## 5. 新增 API 汇总

| 端点 | 方法 | 用途 | 读/写 Base |
|------|------|------|------|
| `/player` | GET | 玩家客户端 HTML 页面 | 无 |
| `/api/player/lookup` | GET | 玩家输入名称后查找信息 | 读：玩家表 + 对战管理 |
| `/api/player/{name}/battle` | GET | 获取战斗状态（玩家视角） | 读：对战管理 + 战斗状态 |
| `/api/player/{name}/available-cards` | GET | 获取可用卡牌列表 | 读：玩家可用牌 |
| `/api/player/select-deck` | POST | 确认 8 张牌选择 | 写：玩家战斗状态 |
| `/api/player/submit-card` | POST | 提交本回合出牌 | 写：回合提交；结算时读写多表 |
| `/api/player/{name}/battle-logs` | GET | 获取战斗日志（玩家视角） | 读：对战记录 |

**总共新增：1 个 HTML 页面 + 6 个 API 端点。** 不需要新增 Base 表或修改现有表结构。

## 6. Base 与网页的数据关系

### 6.1 谁读什么、谁写什么

```
┌──────────────────────────────────────────────────────┐

│                                  │                   │
│  ┌──────────┐               ┌────▼─────┐            │
│  │ 可用卡牌  │←──GET────────│ 过滤查询  │            │
│  │ 勾选区    │               │          │            │
│  └────┬─────┘               └──────────┘            │
│       │                                              │
│  ┌────▼─────┐               ┌──────────┐            │
│  │ 确认牌库  │───POST──────→│ 写牌位    │            │
│  │ [按钮]   │               │ 检查双方  │            │
│  └──────────┘               └────┬─────┘            │
│                                  │                   │
│  ┌──────────┐               ┌────▼─────┐            │
│  │ 出牌提交  │───POST──────→│ 创建提交  │            │
│  │ [下拉+按钮]│              │ 检查双方  │            │
│  └──────────┘               │ →结算     │            │
```

### 6.2 权限隔离

| 数据 | 玩家A能否看到 | 玩家B能否看到 |
|------|------|------|
| 对手名称 | ✅ | ✅ |
| 对手 HP | ✅ | ✅ |
| 对手已出牌（本回合结算后） | ✅ | ✅ |
| 对手牌库（8张） | ❌ 不返回 | ❌ 不返回 |
| 对手性相等级 | ❌ 不返回 | ❌ 不返回 |
| 对手资源状态 | ❌ 不返回 | ❌ 不返回 |

> API 层根据 `player_name` 确定当前玩家的 `side`（A/B），只返回该玩家的详细数据。对手仅暴露名称 + HP + 已出牌（结算后公开信息）。

### 6.3 页面刷新策略

| 场景 | 刷新方式 |
|------|------|
| 本回合已提交、等待对手 | 每 **5 秒**自动轮询 `GET /api/player/{name}/battle`，检测 `my_submitted_this_round` 和 `current_round` 变化 |
| 等待对手确认牌库 | 每 **3 秒**自动轮询，检测 `deck_confirmed` 状态变化 |
| 对战中未出牌 | 不自动刷新（玩家在看卡牌列表） |
| 战斗结束 | 停止轮询，显示最终结果 |
| 手动刷新 | 页面底部「刷新」按钮，随时可用 |

### 6.4 错误处理

| 场景 | 前端处理 |
|------|------|
| 网络断开 | 显示「连接断开，正在重试...」+ 红色状态条 |
| API 返回错误 | 显示错误消息（如「卡牌不合法」「回合已提交」）|
| 战斗不存在 | 显示「暂无进行中的战斗」+ 入口输入框 |
| 对手超时未出牌 | 显示「等待对手中...」+ 已等待时间计数器 |
| 法官强制结束 | 页面刷新后显示「战斗已结束」|
